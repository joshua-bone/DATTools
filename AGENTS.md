# Global Codex working agreements

- Prefer small, incremental changes with clear checkpoints.
- When editing code, preserve existing style and patterns.
- If uncertain, propose 2-3 options with tradeoffs and pick a default.
- Never invent metrics or business facts; request/derive them from data sources or fixtures.
- Always commit and push after every change.
- Before committing, run the formatter needed for touched files so Prettier/format hooks do not fail.
